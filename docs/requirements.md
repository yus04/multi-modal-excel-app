# 要件定義書

## プロジェクト概要
製造業向けExcel作業標準書の画像付き検索システム

## 想定ユーザー/利用環境

### ユーザープロファイル
- **主な利用者**: 現場作業者
- **利用目的**: 手順確認、写真付きで作業内容を理解
- **利用端末**: PC（デスクトップ・ノートPC）

## 機能要件

### 1. 画像同時提示型の検索UI
**FR-001**: システムは検索結果に要約と根拠画像を同時に表示すること
- 要約：手順の概要（最大500文字）
- 画像：関連する作業画像を全て表示
- 参照リンク：元のExcelファイルへのリンク

**FR-002**: システムは元ファイルへのリンクを提供すること
- クラウドストレージ（Azure Blob Storage）上のURLを表示
- 新しいタブで開くことが可能

### 2. 画像＋テキストのハイブリッドRAG検索
**FR-003**: システムはベクトル検索とキーワード検索を組み合わせて実行すること
- ベクトル検索：セマンティック類似性ベースの検索
- キーワード検索：全文検索によるキーワードマッチング
- セマンティックランキング：Azure AI Searchによる結果の再ランク

**FR-004**: システムは画像とテキストを組み合わせた検索を実行すること
- 画像はテキストと紐付けてインデックス化
- 検索結果には関連画像を含む

### 3. Excel→PDF→DI＋MM-LLM構造化
**FR-005**: システムはExcelファイルからレイアウト情報を保持すること
- セル境界の情報を保持
- 画像の配置座標を記録
- 図番号や画像の位置関係を維持

**FR-006**: システムはDocument Intelligence（DI）でレイアウト抽出すること
- ※現在の実装ではopenpyxlを使用
- 将来的にDI統合を推奨

**FR-007**: システムはマルチモーダルLLMで構造化スキーマを定義すること
- GPT-4oを使用して手順を自動構造化
- 手順番号、タイトル、説明、画像を関連付け

## 入力データ範囲

**FR-008**: システムは以下の形式のファイルを受け付けること
- 対象ファイル形式：.xlsx、.xls
- ファイルサイズ：制限なし（ただし、Azure制限に準拠）

## 検索仕様

**FR-009**: システムは手順単位で検索を実行すること
- 検索単位：各作業手順
- 検索結果：手順ごとに1つのカードとして表示

**FR-010**: システムは検索クエリに対してTop-K件の結果を返すこと
- デフォルト：5件
- 最大：20件

## 回答仕様

**FR-011**: システムは以下のフォーマットで回答を返すこと
- 要約：手順の詳細説明
- 画像：関連する全ての画像
- リファレンス：クラウドストレージ上のExcelファイルURL

**FR-012**: 根拠がない時の返し方
- メッセージ：「検索結果が見つかりませんでした。標準書に記載されていない内容の可能性があります。」
- 空の結果リストを返す

**FR-013**: 禁止事項
- 標準書外の推測を行わないこと
- LLMは標準書に記載された内容のみを抽出・要約すること

## 非機能要件

### パフォーマンス
**NFR-001**: 検索レスポンスタイムは3秒以内であること

**NFR-002**: ファイルアップロード処理は5分以内に完了すること

### セキュリティ
**NFR-003**: Azure Key Vaultを使用して資格情報を管理すること（推奨）

**NFR-004**: HTTPS通信を使用すること

**NFR-005**: CORS設定により許可されたオリジンからのみアクセス可能とすること

### 可用性
**NFR-006**: システムは99.9%の可用性を目標とすること

### スケーラビリティ
**NFR-007**: 同時接続ユーザー数50人まで対応すること

## 技術選定

### フロントエンド
- **フレームワーク**: React 18.2
- **言語**: TypeScript
- **ビルドツール**: Vite
- **HTTPクライアント**: Axios

### バックエンド
- **フレームワーク**: FastAPI
- **言語**: Python 3.9+
- **ASGIサーバー**: Uvicorn
- **データバリデーション**: Pydantic

### クラウドプラットフォーム
- **クラウド**: Microsoft Azure

### Azure サービス
- **LLM**: Azure OpenAI Service
  - モデル：GPT-4o (Microsoft Foundry Models gpt-4.5.2 相当)
  - 埋め込み：text-embedding-ada-002
- **検索エンジン**: Azure AI Search
  - セマンティックハイブリッド検索を採用
- **ストレージ**: Azure Blob Storage
- **ドキュメント解析**: Azure AI Document Intelligence（将来実装）

## システム制約

**C-001**: Azure OpenAI Serviceのクォータ制限に従うこと

**C-002**: Azure AI Searchのインデックスサイズ制限に従うこと

**C-003**: Azure Blob Storageの容量制限に従うこと
